# 미리봄 v2 — 표지·썸네일·대시보드 (작업 계획·기준)

원본 기획서(사용자 제공)를 이 저장소 작업 기준으로 통합. 확정 결정 반영.

## 확정 결정 (2026-06-21)
- **보안: 가볍게 간다.** `covers` 버킷 = **공개 읽기(이미지)** + **작성자만 쓰기/삭제**(Storage RLS). 원고 **텍스트는 기존대로 비공개**(RLS·공유토큰). → 표지는 대시보드·뷰어·공유 어디서든 **공개 URL로 바로** 표시(서명URL·Edge Function 미사용). 표지 이미지는 URL 알면 접근 가능하나 경로가 임의 uuid라 사실상 비공개로 취급.
- **자동표지 색**: `hash(id) % 8` (DB 컬럼 없이 결정론적). 같은 책=항상 같은 색.
- 8색 팔레트 HEX·리사이즈 1600/500px·4열·카드클릭=뷰어 = 기획서 기본값(구현 중 미세조정 가능).

## 단계
- **A (프론트, 백엔드 무관)**: 대시보드 표시영역 1080px·가운데정렬, 3:4 세로 카드 4열, **표지없는 자동 썸네일**(8색 결정론 배경 + 가는 선 + 세리프 제목 + 저자, "긴 여름의 끝" 스타일), **롤오버 오버레이 4버튼**(뷰어/썸네일넣기/공유/삭제) + 터치 `⋯` 대체. 기존 상시 삭제버튼(.card-x) 대체. (썸네일넣기는 C단계까지 스텁)
- **B (백엔드)**: 마이그레이션6 — `documents`에 `cover_path`·`cover_thumb_path`(text, nullable) 추가, `covers` 공개읽기 버킷(2MB·jpg/png/webp, 작성자만 쓰기) + Storage RLS. 가벼운 격리 확인(작성자만 업로드/삭제). 사용자 SQL 적용.
- **C (업로드)**: "썸네일 넣기" → 브라우저 리사이즈(표시 ~1600px / 썸네일 ~500px) 2장 업로드 → cover_path/cover_thumb_path 저장. **이미지 썸네일**(object-fit:cover 크롭 + 하단 그라데이션 스크림 + 흰 제목 HTML 오버레이). 교체/삭제 시 Storage 객체까지 제거. 원고 삭제 시 표지 객체도 제거.
- **D (뷰어·공유·정리)**: 뷰어 표지 페이지 **전면화**(여백0·inset:0·시트 라운드에 맞춰 클립). 공유 스냅샷에 표지 표시(공개 URL). 본문 인라인 md 이미지(`![](경로)`)는 **로드 안 하고 플레이스홀더**(파서는 유지, 로드만 차단). 설명서에서 "마크다운 경로 표지" 방법 제거.

## 표지 규칙 (2026-06-22 확정·구현, 커밋 7637d12)
용어: "썸네일" 폐기 → **표지/표지이미지**로 통일.
- **공통**: 표지 쪽은 **항상 1개**(이미지 유무 무관, 뷰어는 표지로 시작). 표지 쪽이 없으면 `ensureCoverPage()`가 최소 표지쪽 삽입(단 `showFront`일 때만). `#` 제목 있으면 표지에 책제목 표시. 본문의 `#`은 원칙적으로 안 쓰되 나오면(단편집) 그대로 본문에 표시(억제 안 함).
- **이미지 있음**(업로드 표지 `STATE.coverUrl`): 이미지를 배경으로 꽉차게(object-fit:cover). 작가가 쓴 `::: 표지` 블록 있으면 그 내용을 **중앙**에 표시(스크림 그라데이션+흰 글씨), 없으면 **이미지만**. (작가가 텍스트 on/off를 `::: 표지` 유무로 제어)
- **이미지 없음**: 배경 = **카드와 같은 색**. 제목(`#`)은 가운데 진하게, 나머지(저자 등)는 **하단에 옅은 다른 색**(.bl-foot, opacity .6).
- **색 통일**: 카드·뷰어·공유 표지색을 **제목 해시**로 일치. 카드 `colorIdx(d.title)`, 뷰어 `deepColor(STATE.colorKey||bookTitle)`에 `colorKey=d.title`(openDoc)·`data.title`(공유) 전달. `COVER_PAL===DEEP_PAL`·해시 동일이라 같은 색 보장. (기존 id해시→제목해시로 바뀌어 기존 카드색 일부 변동)
- 검증: 미리보기 rAF 미발화로 조판 라이브 불가 → CSS 적용·색해시 동일성·구문무오류만 확인. 실제 표지 렌더는 실브라우저(드롭 .md=자동표지 / 대시보드=이미지표지) 확인 필요.

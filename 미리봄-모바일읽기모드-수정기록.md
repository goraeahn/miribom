# 미리봄 — 모바일 읽기 모드 작업 기록 (v1.1)

작성: 2026-06-21 · 대상 파일: `app/index.html` 하나 · 배포: 검증·승인 후 main 머지

## 배경 (왜 고치나)
- 모바일에서 책 '박스'가 화면 가운데 떠서 **위쪽 공간이 크게 낭비**됨.
- 데스크탑 포함 **진행바에 색이 안 차오름**(채움 막대가 안 보임).
- **우하단 쪽번호/라벨이 오른쪽 끝선에 안 붙고 삐뚤어 보임**.

## 결정사항 (논의 결과)
1. **모바일(가로 ≤640px)은 박스 없는 풀화면 페이지넘김 모드.** 페이지 = 화면. 판형 비율 무시.
2. 넘기기: **좌/우 가장자리 탭 + 좌우 스와이프**(기존 유지). **가운데 탭 = 메뉴(도구막대) 보이기/숨기기.**
3. **첫 진입은 메뉴 보이게 시작**(추천안 채택). 가운데 탭 한 번으로 몰입(글자만).
4. 모바일 '보기' 패널: **판형·여백·쪽배치 숨김.** 글꼴(서체·글자크기·행간)·테마만 남김.
5. 모바일에선 **떠 있는 ˄ 접기 버튼 제거**(가운데 탭이 대체). **데스크탑은 ˄ 유지.**
6. 진행바: **화면 맨 아래 얇은 선(2px)을 몰입 중에도 항상 옅게 유지**(플로팅). 숫자(쪽/총쪽)는 메뉴가 떴을 때만.
7. **데스크탑·공유 뷰어의 책 모양/판형/여백/두 면은 그대로**(변경 없음 목표).

## 같이 잡는 버그 (데스크탑 포함 공통)
- **진행바 채움**: 색 막대 `.fill`이 인라인(`<span>`)이라 폭 지정이 무시돼 안 보임 → 박스로 바꿔 해결.
- **우하단 정렬**: `letter-spacing` 꼬리 공백이 마지막 글자 뒤에 붙어 오른쪽 정렬이 어긋남 → 오른쪽 요소에 음수 마진으로 보정.

## 손대는 곳 (app/index.html)
- `sizeBook()` — 모바일이면 "페이지 = 화면" 분기(판형 무시, 양옆 기본 여백 고정).
- CSS `@media(max-width:640px)` — 박스(그림자·테두리·라운드) 제거, 머리말/쪽단 숨김, 스테이지 여백 0, ˄ 숨김, 가운데 탭존 노출, 보기패널 판형·여백 묶음 숨김.
- `.progress` — 채움 버그 수정 + 몰입 시 2px 얇은 선 유지.
- `.page-foot`/`.progress .label` — 오른쪽 정렬 음수 마진 보정.
- 탭존 `#zone-c`(가운데) 추가 → 모바일에서 메뉴 토글. 접기 로직을 `toggleChrome()`로 묶어 ˄·가운데탭 공용.

## 검증 기준
- 데스크탑: 책 카드·판형·두 면·여백 그대로. 진행바 색 참. 우하단 번호 정렬 맞음.
- 모바일(≤640): 박스 없이 글자가 화면을 꽉 채움. 좌/우 탭·스와이프 넘김. 가운데 탭으로 메뉴 토글. 보기패널에 글꼴·테마만. 바닥 얇은 진행선.
